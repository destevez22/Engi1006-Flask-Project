Homework 1

The written portion served as an introduction to basic programming principles, as well
as motivation for creating an algorithm based mindset. The coding portion revealed just how
important this is, as all of our work in python can be described as a bunch of small tasks
written as code that come together to make a program. The us of lists in the second portion
was incredibly useful for exploring syntax and the power of python lists, something we used a lot.

Homework 2

We continued to develop our understanding of basic coding theory with Homework 2, especially
when it comes to time complexity, which is at first a rather weird concept. It was also tied
into the different types of sorting algorithms there are, which can be quite useful. The coding
of this assignment introduced us to simulations and graphing with python, which have evident uses
in many scenarios. We also began to set up our machine learning algorithm and thus learned about
dictionaries and reading csv files.

The first graph shows the trajectory for a projectile in motion. The second graph reveals that the simulated value and expected value do, in fact, diverge, if only slightly in this particular case.

Homework 3

The theory part of this assignment served to further solidify the concepts of time complexity,
as well as introduce the topic of recursion. Recursion is an elegant concept, which often helps
to simplify the solution to certain problems--- it does require caution when it comes to defining
your base case. One part of the coding portion further developed the machine learning algorithm.
The other used recursion and the new topic of numpy arrays to solve a vertical percolation problem.
It's a simple problem in theory but the use of numpy is infinitely useful for many science fields.

The flow of water.. so amazing.

Homework 4

Homework 4 introduced OOP, object oriented programming, a vital part of most programs that involve
end user interaction and an organized model for something. OOP is very useful for allowing objects to
interact with one another and build off of one another, and this homework involved identifying how
exactly this works.

Midterm Assignment

The midterm assignment allowed us to revisit the idea of reading and writing to a file in order to
analyze images. It also allowed us to explore the PPM image format specifically, even if it's not
necessarily the most common image format. The goal of this assignment was to write a program with
three specific functions to interact with or change images. One involved filtering out corrupted
portions of an image using three similar images, another involved applying a grayscale filter to
the image, and the last involved flipping the image horizontally. One of the most important parts
of the midterm assignment was writing the code from scratch!

Mistakes can be truly beautiful sometimes.

Homework 5

In the final homework we completed the machine learning program, wherein we analyzed our data
and produced some plots to go along with it. We were able to explore, pandas, seaborn, and scikit
learn throughout the process, which are all relevant tools for using python for any sort of machine
learning program. The idea was to examine cancer cells and decide whether certain parts of a tumor
were benign or malignant, which has apparent practical applications.

Some scatter plots of the data we ran the algorithm on.

The Best Assignment

The best assignment (at least in my opinion), was the midterm. It certainly took some time and
thinking to complete, but I liked how complete my understanding felt when I finished it(even though
I am sure there are more elegant solutions). This also likely had a lot to do with where we were in
the course at this stage. At first the assignments were confusing and it often felt as though I
didn't have the tools at hand to solve a problem. But by the time I did the midterm assignment
it felt as though things really began to click, and I gained an entirely new appreciation for python.
